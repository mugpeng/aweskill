Vibe-Skills inspired aweskill roadmap · 0511

不要只分发 Skills，要分发可调用的工作流入口

Vibe-Skills 真正值得学习的不是重型 runtime，而是它把“很多技能”包装成一个用户能记住、能调用、能验证的工作方式。aweskill 应该把这个卖点落到 profile、workflow entry、quickstart 和 readiness check 上。

核心判断

aweskill 不应该变成 Vibe-Skills；aweskill 应该成为 Vibe-Skills 这类 super skill / workflow skill 的最佳本地管理层。
当前心智

安装、更新、分组、投影、修复 Skills。

目标心智

把散落 Skills 变成可安装、可复用、可检查的本地工作流系统。

Product Thesis

产品定位：从包管理器到本地 Skill 系统管理器

aweskill 的底层优势仍然是 central store、bundle、projection、doctor、update。不要抛弃这些边界。但对用户表达时，不应该停在“我能管理 skill 文件夹”，而应该表达成“我能把 Agent 的能力组织成可调用的工作流”。 

现在：
aweskill manages local AI agent skills across tools.

更好：
aweskill turns scattered skills into installable, reusable,
checkable workflow systems for local AI agents.

保留边界

不做 runtime authority，不做 workspace memory，不做自动 specialist router。

升级体验

从“linked”升级成“ready for Codex / Claude Code / Cursor”。

Priority 0

引入 profile：用户不想手动拼十个 Skill

`profile` 是 curated bundle 的用户友好入口。它不改变底层模型，只是把 bundle、入口 skill、说明文档和检查规则包装成一个更容易理解的对象。 

aweskill profile list
aweskill profile show coding
aweskill profile install coding --global --agent codex
aweskill profile remove coding --global --agent codex
  • 初始 profiles：`coding`、`research`、`publishing`、`frontend`、`agent-maintenance`。
  • `profile` 复用现有 bundle / projection 逻辑，不创建第二套激活状态。
  • `bundle` 保留为底层精确对象，`profile` 作为高层 UX。
Priority 1

每个 profile 都要有一个入口 Skill

Vibe-Skills 最强的产品符号是 `vibe`。aweskill 的 profile 也应该让用户记住一个入口，而不是面对一串支持技能。

入口 Skill 示例

`coding-workflow`、`research-workflow`、`publishing-workflow`、`frontend-workflow`。

入口 Skill 内容

什么时候用、分阶段选哪些支持技能、非目标、完成前证据要求。

第一版只做 `coding-workflow`。支持技能包括 `brainstorming`、`systematic-debugging`、`test-driven-development`、`verification-before-completion`、`requesting-code-review`、`subagent-driven-development`。
Priority 2

安装完成后必须能检查 readiness

Vibe-Skills 强调 verified delivery。aweskill 不需要验证任务成果，但要验证“这个工作流在目标 Agent 里是否真的可用”。 

aweskill profile check coding --global --agent codex
aweskill profile check coding --project . --agent cursor
aweskill profile check coding --global --agent codex --json
  • 检查 profile、入口 skill、required skills 是否存在。
  • 检查入口和依赖是否投影到目标 Agent。
  • 检查 broken、duplicate、suspicious 状态。
  • 输出 `ready`、`partially ready` 或 `not ready`。
  • 不 ready 时给出下一条建议命令。
profile: coding
entry: coding-workflow
target: codex global
status: ready
Priority 3

做一个 `aweskill quickstart`

Vibe-Skills 用一个入口降低上手成本。aweskill 也需要一个 setup 入口：检测 Agent、推荐 profile、安装、投影、检查、打印使用卡片。 

aweskill quickstart
aweskill quickstart coding
aweskill quickstart coding --global --agent codex
aweskill quickstart --project . --agent cursor
关键约束：先 inspect 再 mutate；目标 agent 或 scope 不明确时再询问；不要默认投影到所有 Agent。
Priority 4

增加 profile template 目录

先把 profile 作为资源模板落地，不急着侵入已有 bundle 模型。这样实验成本低，也方便后续调整。 

resources/profile_templates/coding/
├── profile.yaml
├── README.md
├── usage.md
└── skills/
    └── coding-workflow/
        └── SKILL.md
  • `profile.yaml` 包含 `name`、`entry`、`requires`、`optional`、`tags`、`checks`。
  • 内置入口 skill 可在 profile install 时导入 central store。
  • 支持技能继续通过现有 store / bundle 机制解析。
Priority 5

识别 Vibe-Skills 这类 super-skill 包

aweskill 不必执行第三方 runtime installer，但应该能识别“这不是普通单技能仓库”，并给出保守、清晰的安装建议。 

Detected workflow-style skill package.

Entry:
- vibe

Install options:
- aweskill install <source> --skill vibe
- aweskill install <source> --all

Note:
This package may include its own runtime installer. aweskill can manage projection,
but it will not replace the package's runtime setup unless explicitly supported.
不自动执行第三方 `install.sh`。发现脚本时只展示，并要求用户显式确认。
Priority 6

README 首屏改成三个具体结果

不要第一屏就讲 central store。central store 是架构，不是用户最先感受到的价值。 

aweskill quickstart coding --global --agent codex
aweskill profile install research --global --agent claude-code
aweskill doctor sync --global --agent cursor
  • 给一个 Agent 装上工作流能力。
  • 跨 Agent 复用同一套能力。
  • 修复本地技能混乱状态。
Priority 7

给 Agent 和 CI 准备 JSON 输出

profile 相关命令应该从第一版开始支持 `--json`，方便 Agent 做自动判断。 

aweskill profile list --json
aweskill profile show coding --json
aweskill profile check coding --json
profile entry requiredSkills target status issues suggestedCommands
MVP 1

最小可落地版本

不要一次做完。第一版只验证一件事：用户是否能通过 profile 获得一个可调用、可检查的 coding workflow。

  • 创建 `resources/profile_templates/coding/`。
  • 新增 `coding-workflow` 入口 skill。
  • 实现 `aweskill profile list`。
  • 实现 `aweskill profile install coding --global --agent codex`。
  • 实现 `aweskill profile check coding --global --agent codex`。
  • 安装后打印 usage card。
  • 更新 README 的 workflow-profile 叙事。
aweskill profile install coding --global --agent codex
aweskill profile check coding --global --agent codex

不要做

  • 不要实现 Vibe-like runtime state machine。
  • 不要在 aweskill 里做 workspace memory。
  • 不要做自动 specialist routing。
  • 不要引入隐藏 profile activation registry。

保留 aweskill 的硬边界

  • central store 是唯一事实来源。
  • projection 仍然是 filesystem state。
  • doctor 仍然是 repair path。
  • profile 只是更好的 bundle UX。

一句话方向

面向工作流型 AI Agent 的本地 Skill 系统管理器。

profile entry skill ready check quickstart
